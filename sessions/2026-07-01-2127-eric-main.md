## Task 5: Admin: issue account credit instead of a cash refund (#106)

**Completed:**
- Real-incident-driven feature: a weather cancellation today affected 3 students — one wanted credit instead of a refund, two wanted a transfer. Filed #106/#107/#108 for the three resolutions; closed #108 (course transfer) after design discussion concluded it's just "cancel with full credit, then re-enroll applying that credit" once #106+#107 exist — no separate settlement path needed.
- New `credit_ledger` table (`supabase/migrations/20260702022510_credit_ledger.sql`) — first ledger-style table in the app: immutable rows, balance = `SUM(amount_cents)` per student, never a mutable balance column. RLS: admin INSERT+SELECT only (no UPDATE/DELETE — DB-enforced immutability after a code-review catch), students read their own rows. Second migration extends the pre-existing `payments.status` CHECK constraint to allow `'credited'` (discovered mid-implementation — first `db reset` surfaced it).
- `issueCredit(enrollmentId, courseId, amountCents?)` (`src/actions/enrollments.ts`) — admin-only, modeled directly on the adjacent `processRefund`: same auth gate, same amount validation, but no Stripe call — the school keeps the cash, student gets a non-expiring redeemable balance.
- `enrollment-actions.tsx` — new "Issue Credit & Cancel" dialog next to "Process Refund & Cancel". Admin course detail page's payments query was silently dropping credited payments (only fetched `succeeded`/`refunded`) — fixed alongside the new rendering branch. Admin student detail page shows the balance.
- `docs/DECISIONS.md` DEC-035 records the ledger-over-mutable-column pattern for #107 to extend.
- `supabase/tests/14_credit_ledger.sql` (new) — pgTAP RLS coverage per CLAUDE.md's rule, added after code review flagged it missing.
- `tests/enrollment-credit.spec.ts` (new) — full admin flow + confirms no student-facing credit-request UI exists. Balance asserted as a delta against a baseline read (pw_student2 is a shared fixture across runs) — hit two real flakiness bugs while verifying: a too-tight test timeout, and a not-present locator polling to the ambient timeout when balance is legitimately $0 post-reset (both fixed).
- `npm run build` + lint + full pgTAP suite + targeted Playwright all green.
